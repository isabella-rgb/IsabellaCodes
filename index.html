HTML (Hypertext Markup Language) – Module 1

What is HTML?

HTML is the language used to create and structure webpages.

How I used it:

I used HTML to structure the content of my project and create a clear layout for the user. By using headings, sections, text, and images, I ensured that the information was organised and easy for the user to read and navigate.

Challenges:

One challenge I faced when using HTML was correctly structuring the layout of the website. At first, organising sections and elements in a clear and logical order was difficult. However, through practice and testing, I improved my understanding of HTML structure and was able to create a great layout

Pros

  • Easy to learn and use – simple syntax makes it beginner-friendly.
  • Structures content clearly – allows the user to easily read and navigate the website.
  • Works in all browsers – ensures your website is accessible to everyone.
  • Supports multimedia – you can add images, links, and videos.

    • Cons

  • Limited styling – HTML alone can’t make a site look visually appealing (you need CSS).
  • No interactivity – HTML cannot handle dynamic actions, like button clicks or animations (you need JavaScript).
  • Static content – changes to content require editing the HTML directly.
  • Can be messy if not organised – poorly structured HTML can confuse users and make maintenance difficult.

    • CSS (Cascading Style Sheets) – Module 2

    What is CSS?

    CSS is a language used to style and format the appearance of HTML content on a website.

    How I used it:

    I used CSS to style my project and make it visually appealing for the user. By adding colors, fonts, spacing, and layouts, I improved readability and user experience. CSS allowed me to control the look of sections, images, and icons, and create a consistent design throughout the website.

    Challenges:

    One challenge I faced with CSS was aligning elements and controlling the layout. At first, it was difficult to make sections, images, and icons look organised and balanced. I overcame this by learning about Flexbox and grid layouts, which helped me structure my content neatly for the user.

    Pros

  • Improves user experience – makes the website visually appealing and readable.
  • Controls layout and design – spacing, colors, fonts, and alignment.
  • Separation of content and style – HTML handles structure, CSS handles appearance.
  • Reusable styles – one style can apply to multiple elements, saving time.

    • Cons

  • Can be complex – layouts, positioning, and responsiveness can be tricky to master.
  • Browser differences – some styles may look different on different browsers.
  • Requires practice – without proper knowledge, CSS can be messy and inconsistent.
  • Dependent on HTML structure – poorly written HTML can limit styling possibilities.

    • JS (JavaScript) – Module 3

    What is JavaScript?

    JavaScript is a programming language used to add interactivity and dynamic behaviour to websites.

    How I used it:

    I used JavaScript to add interactivity to my project and improve the user experience. JavaScript allowed me to respond to user actions such as clicks and inputs, and to make the website more dynamic. This helped the website feel more engaging and functional for the user.

    Challenges:

    One challenge I faced when using JavaScript was understanding how functions and events work together. At first, it was difficult to make the website respond correctly to user actions. I overcame this by testing my code, fixing errors, and practising with simple scripts.

    Pros

  • Adds interactivity – responds to user actions like clicks and inputs.
  • Improves user experience – makes websites more engaging and dynamic
  • Works with HTML and CSS – enhances existing content and design.
  • Widely supported – works in all modern browsers.

    • Cons

  • Can be difficult to learn – requires logical thinking and problem-solving.
  • Errors can break features – small mistakes can stop scripts from working.
  • Dependent on the browser – performance can vary between devices.
  • Requires careful testing – to ensure a smooth experience for the user.
